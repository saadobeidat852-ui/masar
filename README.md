# تطبيق منصة مسار — Android (WebView App)

هذا مشروع **Android Studio** كامل وجاهز، يحوّل نسخة `masaar_fixed.html` (نسخة Firebase المستقلة)
إلى تطبيق أندرويد حقيقي بأيقونة واسم خاص بها، قابل للتثبيت كملف APK.

> **ملاحظة مهمة:** تم تجهيز هذا التطبيق بالاعتماد على ملف `masaar_fixed__1_.html` (نسخة Firebase)
> وليس على مجلد `masaar-platform.zip` (نسخة React + Express)، لأن نسخة الـ HTML تعمل بالكامل من
> طرف العميل (Firebase Auth + Firestore) ولا تحتاج خادم Node.js منفصلاً — وهذا يجعلها مناسبة تماماً
> للعمل داخل تطبيق أندرويد. أما نسخة React/Express فتحتاج استضافة خادم Node دائم (لأجل مفتاح
> Gemini API)، وبالتالي لا يمكن تحويلها مباشرة إلى APK دون خادم خلفي مستضاف على الإنترنت أولاً.

## ما الذي يحتويه هذا المشروع؟

- غلاف WebView أصلي بلغة Java (لا React Native ولا Capacitor — تطبيق أندرويد بسيط ومباشر).
- تحميل صفحة `index.html` (المرفقة في `app/src/main/assets/`) عبر نطاق آمن وهمي
  (`https://appassets.androidplatform.net`) باستخدام `WebViewAssetLoader` الرسمي من Google —
  وهذا **ضروري** لأن الصفحة تستخدم وحدات Firebase ES Modules التي يحظرها أندرويد عند التحميل
  المباشر عبر `file://`.
- جسر JavaScript ↔ Java لحفظ ملفات التصدير (مثل تصدير الدرجات إلى Excel عبر `XLSX.writeFile`)
  مباشرة في مجلد التنزيلات على الجهاز.
- دعم اختيار/رفع الملفات (PDF/Word) من داخل التطبيق.
- أيقونة تطبيق مُصمَّمة بنفس تدرّج ألوان هوية مسار (أخضر → أزرق).
- زر الرجوع في أندرويد يعمل كزر "السابق" داخل الموقع.
- رسالة واضحة عند انعدام الاتصال بالإنترنت (التطبيق يحتاج إنترنت لتسجيل الدخول عبر Firebase
  وتحميل الخطوط ومكتبات Chart.js/XLSX من الإنترنت).

## ⚠️ تنبيه مهم بخصوص بيئتي الحالية

أنا لا أملك في بيئة العمل هذه اتصالاً بالإنترنت ولا Android SDK / Gradle مثبَّتين، لذلك **لا
أستطيع تجميع (compile) ملف APK نهائي وتسليمه لك مباشرة الآن**. ما فعلته بدلاً من ذلك هو تجهيز
**مشروع Android Studio كامل وصحيح** يبني بنجاح بمجرد فتحه — لم يتبقَّ سوى خطوة "الضغط على زر
البناء" من جهازك. اخترْ إحدى الطريقتين التاليتين:

---

## الطريقة 1 — البناء محلياً عبر Android Studio (موصى بها، 10 دقائق)

1. نزِّل **Android Studio** مجاناً من: https://developer.android.com/studio
2. افتح Android Studio ← **Open** ← اختر مجلد `masaar-android` (هذا المجلد).
3. انتظر حتى ينتهي **Gradle Sync** تلقائياً (سيُنزِّل Android Studio كل ما يلزم بنفسه: Gradle،
   Android SDK، المكتبات... لن تحتاج تثبيت أي شيء يدوياً، فقط اتصال إنترنت أثناء أول فتح).
4. من القائمة العلوية: **Build ← Build Bundle(s) / APK(s) ← Build APK(s)**.
5. بعد انتهاء البناء ستظهر رسالة "locate" بأسفل الشاشة — اضغطها لفتح مكان ملف
   `app-debug.apk` مباشرة. انسخه إلى هاتفك وثبّته (فعّل "السماح بالتثبيت من مصادر غير معروفة"
   إن طُلب منك ذلك).

### لإصدار نسخة نهائية موقّعة (Release) لرفعها على المتجر أو توزيعها رسمياً:

- **Build ← Generate Signed Bundle / APK ← APK** ← أنشئ مفتاح توقيع جديد (Key store) في
  الخطوات التي تظهر أمامك ← أكمل المعالج. سيُنتج لك ملف APK نهائي موقّع جاهز للتوزيع.

---

## الطريقة 2 — البناء سحابياً عبر GitHub Actions (بدون تثبيت أي برنامج)

أضفت ملف جاهز في `.github/workflows/build-apk.yml` يبني APK تلقائياً على خوادم GitHub المجانية:

1. أنشئ مستودع (repository) جديد فارغ على GitHub.
2. ارفع محتويات مجلد `masaar-android` بالكامل إليه (Push).
3. اذهب إلى تبويب **Actions** في المستودع — سيبدأ البناء تلقائياً خلال دقيقة أو دقيقتين.
4. بعد اكتمال البناء (علامة ✅ خضراء)، افتح التشغيلة (run) ← من أسفل الصفحة قسم
   **Artifacts** ← نزِّل `masaar-debug-apk`.

هذه الطريقة لا تحتاج منك تثبيت أي شيء على جهازك إطلاقاً.

## ملاحظات مهمة قبل التوزيع

1. **مفتاح Firebase مضمَّن بالفعل** في الصفحة (مشروع `massar-edu`) — تسجيل الدخول والتخزين
   سيعملان مباشرة بدون أي إعداد إضافي.
2. **ميزات الذكاء الاصطناعي (OpenAI)**: الصفحة تتيح إدخال مفتاح OpenAI API من داخل إعدادات
   التطبيق نفسه (وليس من السيرفر)، لذا كل مستخدم يُدخل مفتاحه الخاص إن أراد استخدام هذه الميزة.
3. **اسم الحزمة (Package Name)** الحالي هو `com.masaar.app` — إن كنت تخطط للنشر على Google
   Play، يُفضَّل تغييره إلى نطاق فريد تملكه (مثل `com.yourschool.masaar`) قبل التوقيع النهائي،
   ويكون ذلك من ملف `app/build.gradle` (`applicationId`) و
   `app/src/main/java/com/masaar/app/` (إعادة تسمية الحزمة).
4. **الحد الأدنى لإصدار أندرويد المدعوم**: Android 8.0 (API 26) فما فوق — يغطي الغالبية العظمى
   من الأجهزة المستخدمة حالياً.
5. **تحديث محتوى التطبيق لاحقاً**: ببساطة استبدل ملف `app/src/main/assets/index.html` بأي
   نسخة محدّثة من الموقع، ثم أعد البناء — لا حاجة لتعديل أي كود Java.

## هيكل المشروع

```
masaar-android/
├── app/
│   ├── build.gradle                 ← إعدادات التطبيق (اسم الحزمة، الإصدار...)
│   ├── src/main/
│   │   ├── AndroidManifest.xml      ← الصلاحيات واسم التطبيق
│   │   ├── java/com/masaar/app/
│   │   │   └── MainActivity.java    ← كود WebView + جسر التنزيلات + رفع الملفات
│   │   ├── res/                     ← الأيقونات والألوان والتخطيطات
│   │   └── assets/
│   │       └── index.html           ← صفحة منصة مسار (المصدر الفعلي للتطبيق)
├── build.gradle, settings.gradle, gradle.properties
└── .github/workflows/build-apk.yml  ← بناء تلقائي سحابي (اختياري)
```
